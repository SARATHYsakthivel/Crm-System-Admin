Feature: CRM Register and Login Functionality

  Scenario Outline: User Registration Scenarios
    Given user launches browser for registration
    When user opens CRM register page
    And user enters username as "<name>"
    And user enters register email as "<email>"
    And user enters password as "<password>"
    And user enters confirm password as "<confirmPassword>"
    And user enters mobile number as "<mobile>"
    And user clicks register button
    Then user should validate all the fields

    Examples:
      | name            | email                        | password           | confirmPassword    | mobile     | expectedResult          |
      | Parthasarathy S | sarathysakthivel97@gmail.com | Parthasarathy@2026 | Parthasarathy@2026 | 6374060891 | register success        |
      | Parthasarathy   | sarathysakthivel97@gmail.com | Parthasarathy@2026 | Parthasarathy#2025 | 8825528990 | password mismatch error |

  Scenario Outline: User Login Scenarios
    Given user launches browser for login
    When user opens CRM login page
    And user enters login email as "<emailid>"
    And user enters login password as "<loginpassword>"
    And user clicks login button
    Then user should validate all the field

    Examples:
      | emailid                   | loginpassword | expectedResult            |
      | vaishnavibcabhc@gmail.com | Vaishu@123    | login success             |
      | Rohitsharma@gmail.com     | Shana@123     | invalid credentials error |
